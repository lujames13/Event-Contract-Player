# PM-5: Market Implied Probability Calibration 分析

## 1. 調查概述
本分析旨在評估 Polymarket 5m BTC 市場開盤後（約 30-60 秒）的市場價格（Implied Probability）與實際結果（Actual Outcome）之間的校準品質。這個時間段的價格代表了我們實際交易時面臨的真實成本。如果市場定價是高度校準（Well-Calibrated）且無偏的，則單純依賴方向性預測將難以克服手續費成本。

## 2. 數據概覽
- **樣本總數**: 999 個完全匹配的 5m BTC Markets
- **起止時間**: 最近期的歷史市場資料（基於 Gamma API 排列）
- **數據來源**: Gamma API (Metadata) + CLOB API (Price History) + 本地 SQLite `ohlcv` 表 (Volatility)
- **校準價格點**: 開盤後 45 秒最接近的一筆有效成交價（mid-market price），過濾了臨近結算的 certainty 價格。

## 5. 可操作性評估
- **最大 Calibration Deviation**：在足夠樣本下，不論分時或整體定價，實際顯著最大 Deviation 皆 < 3%（統計意義上）。
- **Taker Fee 扣除**：PM-4 調查指出 taker 成本和滑價是客觀存在的。若我們盲目做對立交易，不到 2% 的 Deviation 將被輕易吞噬（即 net edge <= 0）。
- **如果用 Maker order**：即使達到 0 fee 並能取得 Maker Rebate，其極薄差額代表必須依靠極大規模才能獲利，承擔流動性和跨度風險。

## 6. 結論

**結論：🔴 Market is Well-Calibrated (Efficient)**

**總結說明：**
即使我們更正了採樣點，精確採用了開盤後 30-60 秒這一「真實可操作區間」的中段價格作為 Implied Probability，並帶入波動率與時段交叉分析：
1. **整體校準依然良好**：40-60% 最頻繁出現的價格區間內，Deviation 只有 1.1% - 1.7%，完全未突破統計信心區間（~3.8-5.6%）。市場不存在穩定超過 fee 的系統性定價錯誤。
2. **缺乏明顯結構差異**：時段與波動性的 Sub-group 並未分離出偏差 >10% 的高確定性獲利池。
3. **沒有「簡單套利」的空間**：對於系統而言，**純方向預測無額外 edge**。要在此環境獲勝，不能依賴市場失靈，而是必須讓我們的私有模型（CatBoost/LGBM）產生大於市場共識的 Predictive Accuracy（要求在 PM-8 檢驗是否優於市場價格）。
